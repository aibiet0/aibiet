Article text starts here. First paragraph is in bold and have drop cap letter. Achieved by putting text in class="fy2-article-text". If drop cap and bold text is not necessary, add an empty paragraph at the beginning of the text.

Editors can add as many paragraphs as they want. All text is entered into a single field "Initial Section" of the article page.

Second part of the article has a floating object. The floating object should come first. And then the text that floats around the object. Floating effect is achieved by wrapping the object inside class="fy2-float-right". This class is generated by Floating Container Component. The component has the following parameters:

  • Float right choice. To stop floating remove the container (permanently) or select none.
  • Width percentage of the column
  • Right margin: How much the object floats to the right side of the column
  • Float left choice. To stop floating remove the container (permanently) or select none.
  • Left margin: How much the object floats to the left side of the column

Normal text can still appear here. The text that flows around the object is stored in another field of the article page.

This text is inside a blockquote. The quotation look-and-feel are defined by class="mod mod-Content_quotation" and blockquote and p class="copy-quotation". The floating effect itself is define by class="fy2-float-left" in the parent div

This is third section text that the editor can enter.

Two components on top of each other.

For illustration purpose, another floating container can be added. It is configured to float to the right, and contains an image and a text quotation instead of a video player. The editor can select the type of component to add. The component(s) inside the floating container can be complex.

The approach is scalable. It requires just adding one more placeholder and one extra field for the Article Page. Initially we can limit the design to 2 floating object per pages which probably can cover most of articles. But it is easy to extend with new fields and placeholders if there is such a necessity.

Final text paragraph in the article. Again the editor can place as much text here as desirable. The text is stored in a Sitecore field on the page. After the final text block come the text navigation components "Back to Magazine" and "Back to Top".
